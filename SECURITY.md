# Security Policy

## Supported versions

当前维护最新稳定版本；必要时也会修复最新预发布版本中的安全问题。

## Reporting a vulnerability

请不要在公开 Issue 中披露可能导致 Cursor 数据损坏、隐私泄漏或任意代码执行的问题。请通过 GitHub Security Advisories 的 “Report a vulnerability” 私下提交报告，并附上：

- 受影响版本和操作系统；
- 最小复现步骤；
- 已脱敏的 schema、错误和行为描述；
- 可能的数据影响。

请勿附带真实数据库、对话正文、密钥、用户名或完整工作目录路径。
